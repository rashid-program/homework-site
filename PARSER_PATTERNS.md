# Паттерны и ошибки парсера parse-tasks.mjs

## ОШИБКА 1: √ с дробью под корнем

**Что приходит из PDF:**
```
√
9
4
.
```

**Что генерировал парсер (неверно):**
`\frac{√}{9} 4` → PNG: `√/9 · 4`

**Правильный LaTeX:**
`\sqrt{\dfrac{9}{4}}`

**Правило:** Если токен `√` стоит первым, а следующие два токена — числа без операторов → это `\sqrt{\dfrac{числитель}{знаменатель}}`.

---

## ОШИБКА 2: Unicode-переменные из PDF

**Символы из PDF (не обычные латинские!):**
- `푥` (U+D465) → должно стать `x`
- `푦` (U+D466) → должно стать `y`
- `푎` (U+D44E) → должно стать `a`
- `푏` (U+D44F) → должно стать `b`

**Что происходило:** KaTeX не знает эти unicode-символы → рендерится как кракозябры.

**Исправление в `escLaTeX`:**
```js
.replace(/푥/g, 'x').replace(/푦/g, 'y')
.replace(/푎/g, 'a').replace(/푏/g, 'b')
```

---

## ОШИБКА 3: Степени (показатели степени)

**Что приходит из PDF:**
```
16푥
4
푦
6
```
Это означает `16x^4 y^6` — следующий числовой токен это степень.

**Что генерировал парсер (неверно):**
`\dfrac{16x}{4} y 6` → дробь вместо степени

**Правильный LaTeX:** `16x^{4}y^{6}`

**Правило:** Если токен заканчивается на переменную (a,b,x,y), а следующий — одиночное целое число → это степень `^{...}`.

---

## ОШИБКА 4: √ с variation selectors

PDF содержит `√` с разными variation selectors (U+FE00–U+FE03):
- `√︀` `√︂` `√︃` — всё это один и тот же знак корня

**Правило:** Стрипать variation selectors перед обработкой (уже делается на уровне строк, нужно также в токенах).

---

## ОШИБКА 5: Текст "при x=... и y=..." в конце

**Что приходит из PDF:**
```
при푥= 6и푦= 2.
```
**Правило:** Строку вида `при...=` отделять как текстовую часть, не LaTeX.

Задание 6 содержит только:
- Дроби → `\dfrac{}{}` — работает
- Простая арифметика +, −, ·, × — работает
- Десятичные запятые → `{,}` — работает

## Задание 8 — статус ❌ ТРЕБУЕТ ПЕРЕГЕНЕРАЦИИ

Содержит сложные типы:
1. `√(дробь)` — неверно
2. `√(многочлен со степенями)` — неверно (переменные как кракозябры)
3. `a^n`, `x^4 y^6` — неверно (интерпретируются как дроби)
4. `(2√6)^2` — частично неверно
5. `5^{-7}·(5^5)` — неверно

---

## Алгоритм исправления parse-tasks.mjs

### 1. Нормализация строк (добавить в начало обработки):
```js
function normalizeLine(s) {
  return s
    .replace(/[\uFE00-\uFE0F]/g, '')  // strip variation selectors
    .replace(/푥/g, 'x').replace(/푦/g, 'y')
    .replace(/푎/g, 'a').replace(/푏/g, 'b');
}
```
Вызывать после `.trim()` в обработке `lines`.

### 2. В `escLaTeX` добавить нормализацию:
```js
function escLaTeX(s) {
  return s
    .replace(/[\uFE00-\uFE0F]/g, '')
    .replace(/푥/g, 'x').replace(/푦/g, 'y')
    .replace(/푎/g, 'a').replace(/푏/g, 'b')
    .replace(/·/g, '\\cdot ')
    .replace(/−/g, '-')
    .replace(/×/g, '\\times ')
    .replace(/(\d),(\d)/g, '$1{,}$2');
}
```

### 3. В `toLatex` — обработка `√`:
Вместо того чтобы `√` попадал как обычный токен, добавить в начало цикла:
```js
// Корень
if (cur === '√') {
  const next1 = exprLines[idx + 1];
  const next2 = exprLines[idx + 2];
  const isPureFraction =
    next1 && next2 &&
    /^[\d.,]+$/.test(next1) && /^[\d.,]+$/.test(next2);

  if (isPureFraction) {
    // √(числитель/знаменатель)
    latex += `\\sqrt{\\dfrac{${escLaTeX(next1)}}{${escLaTeX(next2)}}}`;
    idx += 3;
  } else if (next1) {
    // √(выражение)
    latex += `\\sqrt{${escLaTeX(next1)}}`;
    idx += 2;
  } else {
    latex += '\\sqrt{}';
    idx++;
  }
  continue;
}
```

### 4. Степени — токен вида `число + переменная` с последующим числом:
После нормализации unicode, в `toLatex`:
```js
// Если cur содержит переменную в конце, следующий токен — целое число → степень
if (/[a-zA-Z]/.test(cur) && idx+1 < exprLines.length) {
  const next = exprLines[idx+1];
  if (/^\d+$/.test(next) && !STANDALONE_OPS.has(next)) {
    latex += escLaTeX(cur) + `^{${next}}`;
    idx += 2;
    continue;
  }
}
```
